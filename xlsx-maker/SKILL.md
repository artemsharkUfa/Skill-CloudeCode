# XLSX Maker

Скилл для создания Excel-таблиц через openpyxl с формулами, стилями и графиками.

## Workflow

### Шаг 1: Сбор требований

Узнать у пользователя:
1. Тип таблицы (финмодель / дашборд / план / трекер / отчёт / произвольная)
2. Какие данные — структура: листы, колонки, строки
3. Нужны ли формулы, графики, условное форматирование
4. Стиль оформления (корпоративный / нейтральный)

### Шаг 2: Планирование структуры

Определить:
- Список листов (sheets)
- Колонки с типами данных (текст, число, дата, формула)
- Формулы (SUM, AVERAGE, VLOOKUP, IF и т.д.)
- Графики (если нужны)

Показать план пользователю для подтверждения.

### Шаг 3: Генерация

Запустить скрипт:
```bash
python3 ~/.claude/skills/xlsx-maker/scripts/create_xlsx.py \
  --output "~/outputs/table-name.xlsx" \
  --structure structure.json
```

Или собрать таблицу напрямую через openpyxl в inline-коде.

**Принципы генерации:**
- Заголовки: жирный шрифт, цветной фон, замороженная строка
- Ширина колонок: автоподгон по содержимому
- Числа: числовой формат с разделителем тысяч
- Валюта: формат #,##0.00 ₽ или $#,##0.00
- Даты: DD.MM.YYYY
- Формулы: именованные диапазоны где уместно
- Условное форматирование: зелёный/красный для отклонений

### Шаг 4: Валидация

Проверить:
- [ ] Файл создан и открывается
- [ ] Все формулы корректны (нет #REF!, #NAME?)
- [ ] Форматирование единообразное
- [ ] Ширина колонок достаточная
- [ ] Данные на своих местах

### Шаг 5: Выдача

Сообщить путь к файлу, описание листов и ключевых формул.
Файлы сохраняются в `~/outputs/`.

## Связка с pptx-maker

Если после создания таблицы пользователь просит презентацию:
1. Указать путь к .xlsx файлу
2. pptx-maker прочитает данные и визуализирует в слайдах

## Обработка особых случаев

**Пользователь даёт сырые данные (текст, CSV, JSON):**
Парсить данные, структурировать в таблицу, подтвердить у пользователя.

**Пользователь хочет редактировать существующий .xlsx:**
Прочитать через openpyxl, показать текущую структуру, внести изменения.

**Большой объём данных (1000+ строк):**
Использовать openpyxl в write-only mode для производительности.

## Частые ошибки — НЕ допускать

1. Формулы с захардкоженными значениями вместо ссылок на ячейки
2. Текст в ячейках с числами (нарушает формулы)
3. Слитые ячейки без необходимости (ломают сортировку)
4. Отсутствие заморозки заголовков
5. Слишком узкие колонки (данные обрезаются)
